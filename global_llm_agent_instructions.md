# Agent Guidelines

## Code Generation
- Write self-documenting code. 
- Avoid comments unless the logic is complex or counter-intuitive. 
- Focus on clear variable/function naming over explanatory text.

## Git Commit Guidelines
When generating commit messages, always follow the 50/72 Rule:
- Subject: 50 characters or less, imperative mood, no period.
- Separator: One blank line between subject and body.
- Body: Wrapped at 72 characters per line.
